Passing Properties to DITA-OT

How to pass properties to DITA-OT transformations.

Properties are passed to DITA-OT as Ant properties, controlling various aspects of the transformation process.

Groovy DSL Syntax

Use the properties closure:

dita {
    ditaOt '/path/to/dita-ot'
    input 'guide.ditamap'
    transtype 'html5'

    properties {
        property(name: 'processing-mode', value: 'strict')
        property(name: 'args.css', value: 'custom.css')
        property(name: 'args.cssroot', value: "${projectDir}/css")
    }
}

Kotlin DSL Syntax

Use the properties lambda:

tasks.named<com.github.jyjeanne.DitaOtTask>("dita") {
    ditaOt(file("/path/to/dita-ot"))
    input("guide.ditamap")
    transtype("html5")

    properties {
        "processing-mode" to "strict"
        "args.css" to "custom.css"
        "args.cssroot" to "$projectDir/css"
    }
}

Direct API (Recommended for Configuration Cache)

For best Configuration Cache compatibility, use ditaProperties.put():

tasks.named<com.github.jyjeanne.DitaOtTask>("dita") {
    ditaProperties.put("processing-mode", "strict")
    ditaProperties.put("args.css", "custom.css")
}

Common Properties

Property Transtype Description
processing-mode All Processing mode: strict, lax, skip
args.css HTML CSS file to include
args.copycss HTML Copy CSS to output (yes/no)
nav-toc HTML5 TOC navigation: none, partial, full
outputFile.base PDF Output PDF filename (without extension)
args.chapter.layout PDF Chapter layout: MINITOC, BASIC
