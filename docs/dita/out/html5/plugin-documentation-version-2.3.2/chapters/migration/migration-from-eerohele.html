Migration from eerohele/dita-ot-gradle

Upgrade guide for users of the original plugin.

This plugin is a fork of eerohele/dita-ot-gradle with additional features and improvements.

Why Migrate?

  • Full Configuration Cache Support: 77% faster incremental builds
  • Gradle 9 Compatibility: Works with latest Gradle versions
  • Active Maintenance: Regular updates and bug fixes
  • Improved Documentation: Comprehensive examples and guides

Migration Steps

  1. Update Plugin ID:

    // Before
plugins {
    id 'com.github.eerohele.dita-ot-gradle' version 'X.X.X'
}

// After
plugins {
    id 'io.github.jyjeanne.dita-ot-gradle' version '2.3.2'
}

  2. Review Task Configuration: Most configurations are compatible. Update any deprecated options.

  3. Enable Configuration Cache: Add to gradle.properties:

    org.gradle.configuration-cache=true

  4. Test Your Build: Run your existing build to verify compatibility.

Breaking Changes

There are no breaking changes from the original plugin. All existing configurations should work without modification.

New Features Available

After migration, you can take advantage of:

  • ditaProperties.put() for Configuration Cache compatibility
  • Provider API support for lazy evaluation
  • Improved error messages and diagnostics
  • Better Kotlin DSL support
