Product Version Documentation

Generate version-specific documentation using DITA profiling.

The examples/version-docs project demonstrates conditional content processing for different product versions.

Quick Start

cd examples/version-docs

# Build all versions
./gradlew buildAll

# Build specific version
./gradlew buildV1
./gradlew buildV2
./gradlew buildV3

DITAVAL Filtering

Create version-specific filters in filters/:

<!-- filters/v2.ditaval -->
<val>
  <prop action="include" att="product" val="v2"/>
  <prop action="include" att="product" val="v2-v3"/>
  <prop action="exclude" att="product" val="v1"/>
  <prop action="exclude" att="product" val="v3"/>
</val>

Conditional Content

Mark content with product attributes:

<!-- Content for all versions -->
<p>This feature is available in all versions.</p>

<!-- Content for specific version -->
<p product="v3">New in v3: Advanced analytics dashboard.</p>

<!-- Content for multiple versions -->
<p product="v2-v3">Available in v2 and v3.</p>

Task Configuration

tasks.register<DitaOtTask>("buildV2") {
    input(file("content/guide.ditamap"))
    output(layout.buildDirectory.dir("output/v2"))
    transtype("html5")
    filter(file("filters/v2.ditaval"))
}

Best Practices

  • Use consistent profiling attribute values (e.g., v2-v3 for shared content)
  • Keep version-specific content minimal
  • Test all version outputs to ensure correct filtering
  • Document profiling conventions in your style guide
